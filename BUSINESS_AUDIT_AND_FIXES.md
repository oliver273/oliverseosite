## ‚ùå PROBLEM 2: NO CONTRACT/AGREEMENT

**The Issue:**
You have no written contract. This means:
- Client can refuse to pay
- Scope creep ("just one more thing")
- No legal protection
- "That's not what I asked for"

**The Fix:**
Use this simple contract template:

```
WEB DESIGN AGREEMENT

Client: [Business Name]
Designer: OliverSEO
Date: [Date]

PROJECT SCOPE
- Website Type: [Essential/Professional/Premium]
- Number of Pages: [X]
- Features: [List features]
- Timeline: [X] business days

PRICING
- Total: $[X]
- Deposit: $[X] (50%) - Due before work begins
- Final: $[X] (50%) - Due before site goes live

WHAT'S INCLUDED
- Custom design
- Mobile-friendly
- [X] rounds of revisions
- Basic SEO setup
- Training on how to use site

WHAT'S NOT INCLUDED
- Content writing (unless purchased separately)
- Photography (client provides)
- Ongoing maintenance (available separately)
- Domain registration (client responsibility OR billed separately)

REVISIONS
- A revision = changes to design/layout
- Text/image swaps are unlimited during project
- After launch: $100/hour

OWNERSHIP
- Client owns final website after full payment
- Designer retains right to use in portfolio

APPROVAL
By paying the deposit, client agrees to these terms.

Deposit Link: [Square Invoice URL]
```

**Send this as PDF or text before they pay deposit.**

## ‚ùå PROBLEM 5: NO HOSTING SETUP GUIDE

**The Issue:**
How exactly do you set up Cloudflare hosting for a client?

**The Fix:**

### For YOUR Sites (Cloudflare Pages - Free)
1. Push code to GitHub
2. Connect GitHub to Cloudflare Pages
3. Auto-deploys on push

### For CLIENT Sites

**Option A: Client Buys Domain, You Host (Recommended)**
1. Client buys domain at Cloudflare ($10-15/year)
2. You add their site to YOUR Cloudflare account
3. You maintain control
4. Bill them $49/month hosting

**Option B: Client Owns Everything**
1. Client creates Cloudflare account
2. Client buys domain
3. You build site, push to their GitHub
4. You set up Cloudflare Pages for them
5. They own everything, you lose control

**Option C: You Own Domain, Client Pays Monthly**
1. YOU buy domain for client
2. YOU host on your Cloudflare
3. Bill domain cost + hosting monthly
4. You control everything (easiest to manage)

**Best Practice:**
- Use Option A or C
- Keep control of hosting
- Monthly payments = recurring revenue

## ‚ùå PROBLEM 6: WHAT COUNTS AS A "REVISION"?

**The Issue:**
"2 rounds of revisions" - but what IS a revision?

**The Fix:**

### A Revision IS:
- Changing the layout
- Different color scheme
- Moving sections around
- Adding new sections
- Major design changes

### A Revision IS NOT:
- Fixing typos
- Swapping photos
- Updating text content
- Minor adjustments
- Bug fixes

**What to Tell Clients:**
```
"You get [X] rounds of revisions for design changes. 
Small stuff like typos and photo swaps are unlimited - just ask!"
```

## ‚ùå PROBLEM 7: WHAT IF THEY DON'T PAY FINAL 50%?

**The Issue:**
You do the work, they see the preview, they ghost you.

**The Fix:**

### Prevention:
1. **Never give live site access until paid**
   - Preview on a staging URL (e.g., clientname.oliverseo.com)
   - They can see it, but they don't own it

2. **Use Watermark on Preview**
   - Add "PREVIEW - NOT FINAL" watermark
   - Remove when paid

3. **Set Expectations Early**
   - "Site goes live after final payment"
   - In your contract

### If They Ghost:

**Message 1 (Day 1):**
```
Hey [Name]! Your site is ready for launch. 

Once you pay the final $[X], I'll:
- Remove the preview watermark
- Connect it to your domain
- Make you live!

Here's the invoice: [link]
```

**Message 2 (Day 3):**
```
Just following up - did you get my last message?

Your site is ready to go. Just need that final payment 
and you'll be live today!
```

**Message 3 (Day 7):**
```
Hi [Name], 

I haven't heard back about the final payment. 

I'll hold your project for another week, but after that 
I'll need to move on to other clients.

Let me know if there's an issue I can help with.
```

**Message 4 (Day 14):**
```
Final notice: Your website project will be archived 
tomorrow if I don't receive payment.

You can restart the project anytime by paying the 
remaining balance.

Thanks for understanding.
```
