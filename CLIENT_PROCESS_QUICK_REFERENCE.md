## üöÄ STAGE-BY-STAGE CHECKLIST

### STAGE 1: Initial Contact
- [ ] Respond within 2 hours
- [ ] Send discovery questions
- [ ] Add to client tracker
- [ ] Set follow-up reminder

**Key Question:** "What's your main goal for this website?"

---

### STAGE 2: Discovery
- [ ] Send discovery questionnaire
- [ ] Schedule call (if needed)
- [ ] Gather all requirements
- [ ] Determine best package

**Key Question:** "What would make this project a success?"

---

### STAGE 3: Proposal
- [ ] Create custom proposal
- [ ] Include timeline & pricing
- [ ] Send within 24 hours
- [ ] Follow up if no response (2 days, 5 days)

**Key Question:** "Does this proposal work for you?"

---

### STAGE 4: Agreement & Payment
- [ ] Send 50% deposit invoice
- [ ] Wait for payment
- [ ] Send kickoff email once paid
- [ ] Request all materials

**Key Question:** "Ready to get started?"

---

### STAGE 5: Kickoff
- [ ] Collect all materials
- [ ] Create materials checklist
- [ ] Set project timeline
- [ ] Start design mockup

**Key Question:** "Do you have all content ready?"

---

### STAGE 6: Design & Development
- [ ] Create design mockup (Day 1-2)
- [ ] Get design approval
- [ ] Build full website (Day 3-4)
- [ ] Use quality checklist
- [ ] Send daily updates

**Key Question:** "Do you approve this design?"

---

### STAGE 7: Revisions
- [ ] Send for review
- [ ] Collect feedback
- [ ] Make changes (within 24-48 hours)
- [ ] Get approval
- [ ] Charge for extra revisions if needed

**Key Question:** "What needs to change?"

---

### STAGE 8: Delivery & Launch
- [ ] Request final payment (50%)
- [ ] Launch website once paid
- [ ] Test everything
- [ ] Send launch email
- [ ] Deliver all files

**Key Question:** "Ready to launch?"

---

### STAGE 9: Follow-up
- [ ] Check in 1 week after launch
- [ ] Request testimonial (2 weeks)
- [ ] Offer maintenance/upsells (1 month)

**Key Question:** "How's the website working for you?"

## üö® RED FLAGS TO WATCH FOR

- **Budget too low:** Still help, adjust package
- **Unrealistic timeline:** Offer rush option or adjust
- **Vague requirements:** Ask more questions
- **No materials:** Offer to create them
- **Unlimited revisions request:** Set limits upfront
- **Scope creep:** Charge for additions

## üí° QUICK TIPS

1. **Respond fast** (within 2 hours)
2. **Communicate daily** (even if just "working on it")
3. **Set clear expectations** (timeline, revisions, scope)
4. **Get approval before building** (design mockup first)
5. **Don't launch until paid** (final payment first)
6. **Follow up always** (after launch, for testimonials)
7. **Document everything** (emails, notes, files)
